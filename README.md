# 船舶制造/修理质量-缺陷-故障知识库 Agent Demo

纯前端静态/半动态演示页面，面向企业后台/数据治理平台风格。Demo 不接真实后端、不接真实大模型，使用 `public/app.js` 内置模拟数据完成场景切换、治理前后对比、标准化结果、知识图谱关系、Agent 输出和评测结果展示。

## 本地预览

```bash
cd public
python3 -m http.server 3005
```

访问：

```text
http://localhost:3005/
```

## 在线部署

本项目已整理为 GitHub Pages 静态站点，部署内容来自 `public/` 目录。

推送到 GitHub 后，在仓库的 `Settings → Pages` 中将 Source 设置为 `GitHub Actions`。每次推送到 `main` 分支都会自动发布。

上线后地址通常为：

```text
https://你的GitHub用户名.github.io/仓库名/
```

## Demo 范围

- 统一平台，不是三个独立系统
- 一个治理编排 Agent/总控 Agent，负责识别用户问题并调度子 Agent
- 三个业务子 Agent：质量异常 Agent、现场缺陷 Agent、设备故障 Agent
- 左侧导航为真实页面切换：总览、资产目录、字典主数据、治理任务流、知识图谱、Agent 应用、评测审计
- 数据资产目录支持模拟历史文件上传和新增数据采集表单，提交后生成治理任务
- 共用治理流程：历史数据接入 → 数据资产登记 → 字段抽取 → 标准字典匹配 → 主数据对齐 → 数据质量校验 → 人工复核 → 入知识库/知识图谱 → Agent 调用 → 反馈回流
- 数据治理任务流支持点击任务，并逐步执行字段抽取、标准化匹配、主数据对齐、质量校验、人工确认、入知识库/图谱
- 标准字典/主数据支持新增别名映射
- 知识库/知识图谱会根据治理任务入库结果动态新增案例和三元组
- Agent 应用支持点击 Agent 查看详情，并运行三 Agent 协同演示
- 总览展示治理前后价值指标，包括字段完整率、标准命中率、主数据对齐和可引用证据
- 顶部增加全局角色按钮：集团、子公司、一线；不同角色看到不同看板指标和页面提示
- 总览增加科技感态势区，使用生成的船舶数据治理视觉资产 `public/assets/ship-governance-command.png`
- 总览增加三类角色视角：集团治理驾驶舱、二三级公司工作台、一线轻量采集助手
- 集团视角展示子公司治理成熟度、资产登记、标准覆盖、主数据对齐和价值闭环
- 二三级公司视角展示历史批次治理、自动清洗/人工复核比例、规则复用和价值闭环
- 一线视角展示拍照/上传/一句话采集、少填字段、标准推荐和即时业务收益
- 总览提供「一键演示治理闭环」，自动跑通压载水系统联调异常主线
- 数据治理任务流增加血缘追溯、规则校验、业务影响、抽取证据、质量规则和反馈回流
- 治理任务详情展示治理前后对比：自由文本/不可关联 → 标准字段/主数据/图谱/Agent 可引用
- 知识库页面展示案例库、标准库、模式库、整改措施库、主数据实体和专家经验库
- 知识图谱页面内嵌生成规则说明：实体生成、标准化、关系生成、入图谱门槛、置信度和证据链规则
- 知识图谱页面提供当前任务的生成过程预览，展示从原始记录到候选三元组、入图判断和 Agent 可引用状态
- 数据治理任务流内嵌治理作业规则，强调原始记录冻结、标准化依据、主数据置信度和入库门禁
- Agent 应用内嵌运行规则，说明先治理后调用、总控拆分、子 Agent 边界和可审计输出要求
- Agent 详情展示可信知识调用过程：检索案例、命中实体、引用标准、不确定性和回流机制；编排结果以智能业务处置单呈现
- 评测与审计展示闭环反馈记录，用于体现专家采纳、入库和评测样本沉淀
- 用户新增任务、上传记录、字典别名、知识库案例会保存到 localStorage，刷新后保留
